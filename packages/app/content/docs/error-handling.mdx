## Error boundaries you should expect

In practice, Agent Layer integrations should treat errors in three groups:

1. Authentication failures
2. Network or transport issues
3. Job execution or billing failures

## Minimal pattern

<CodeBlock
  language="typescript"
  title="error-handling.ts"
  code={String.raw`try {
  const result = await client.embeddings.create({
    model: "agent-embed-1",
    input: "Decentralized AI infrastructure"
  });

  return result;
} catch (error) {
  const message =
    error instanceof Error ? error.message : "Unknown Agent Layer error";

  console.error(message);
  throw error;
}`}
/>

## Streaming failures

When you stream chat completions, handle both:

- failure to create the job
- failure after the WebSocket has already opened

<CodeBlock
  language="typescript"
  title="stream-guard.ts"
  code={String.raw`const stream = await client.chat.completions.create({
  messages: [{ role: "user", content: "Hello" }],
  stream: true
});

try {
  for await (const chunk of stream) {
    process.stdout.write(chunk.choices?.[0]?.delta?.content ?? "");
  }
} catch (error) {
  console.error("Stream failed:", error);
}`}
/>

## UX guidance

- Surface a concise user-facing message.
- Keep full diagnostic detail in logs.
- Make retries explicit.
- Preserve partial streamed output when useful.

<Callout>
  Do not assume a balance precheck eliminates billing-related failures. Always treat the final execution result as authoritative.
</Callout>
