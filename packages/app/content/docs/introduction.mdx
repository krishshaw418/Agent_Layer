## Build on a network, not a single inference box

Agent Layer is a decentralized AI infrastructure that accepts application requests, distributes jobs to nodes, and returns results through a unified SDK surface.

- Applications send chat, embeddings, and account-related operations.
- The network routes each job according to availability and strategy.
- Nodes execute work and stream results back to the caller.
- Vault + Escrow handle token-native payment settlement.

## Mental model

Think about Agent Layer as three layers:

1. A developer API layer that looks familiar.
2. A routing layer that decides where work should run.
3. A settlement layer that accounts for compute usage with tokens.

## Why teams use it

- Reduce dependence on a single inference provider.
- Match request routing to product goals such as lower cost or lower latency.
- Stream responses back in real time for chat experiences.
- Build wallet-aware billing flows into the same platform.
