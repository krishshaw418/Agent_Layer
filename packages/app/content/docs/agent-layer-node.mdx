## Prerequisites

Before running the setup script, install these dependencies:

- **Git** — [Download Git](https://git-scm.com/downloads)
- **Bun v1.3.6** — Install with:

```bash
curl -fsSL https://bun.sh/install | bash -s "bun-v1.3.6"
```

Verify with:

```bash
bun --version
```

- **Ollama** — [Download Ollama](https://ollama.com/download) for local model execution

## 1. Download and run `setup.sh`

1. Download `setup.sh` from [agentlayerai.vercel.app](https://agentlayerai.vercel.app/)
2. Open a terminal in the folder where you downloaded it
3. Make it executable and run it:

```bash
chmod +x setup.sh
./setup.sh
```

The script will:

- Clone the `node_server` into a local `Agent_Layer/` directory
- Write `default.config.json` automatically
- Prompt you for the values that belong in `manual.config.json`

### `MODEL` - Ollama model name

Set `MODEL` to any Ollama model you want to run locally, based on the hardware available on your machine. Pull the model first, then use the same model name in the config.

```bash
ollama pull <your-model-name>
```

For model selection, installation help, and usage details, refer to the [Ollama documentation](https://ollama.com/docs).

## 3. Server starts automatically

Once all values are entered, the script will start the node server for you:

```bash
bun dev
```

The server should come online on port 8080. Keep that terminal open; closing it stops the node.

To restart it later:

```bash
cd Agent_Layer
bun dev
```

## What gets created

After the script finishes, your `Agent_Layer/` directory will look like this:

```text
Agent_Layer/
├── default.config.json
├── manual.config.json
└── ... node_server files
```
